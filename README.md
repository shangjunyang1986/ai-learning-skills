# ai-learning-skills

> 用 AI 把一个 **GitHub 项目 / 论文 / 技术书 / 陌生领域**，快速变成一份**带图、可交互、能离线打开和分享**的学习网页。
> 为**快速上手、个人学习**而生——刻意轻量：一个自包含 HTML，双击即开，可单独用、也可合并用。

![对 openai/whisper 跑一句话生成的学习页](docs/images/03-single-repo-whisper.png)

<p align="center"><sub>↑ 只说一句「帮我学习 github.com/openai/whisper」，自动生成的学习页：左目录右内容、官方架构图已下载到本地、真实星标、图片可点击放大</sub></p>

## 你是不是也是「小杨」

> 老板：“**小杨**，我在网上看到这个 GitHub 项目不错，你去学习学习，咱们用起来。”
>
> 老板：“**小杨**，你看这个组织不错，最近做了很多事，咱也能做，做起来。”
>
> 老板：“**小杨**，竞争对手刚开源了个东西，你扒一扒，看咱能不能用、能不能超。”
>
> 老板：“**小杨**，群里转的这篇 arXiv 你瞅瞅，靠不靠谱、能不能落地。”
>
> 老板：“**小杨**，我觉得这个领域挺有前景，决定往这儿做一做，你研究研究，做个验证。”
>
> 老板：“**小杨**，这本书讲的技术正好我们要用，你先吃透，回头给大家讲讲。”

而小杨的内心 OS 永远是同一句：

> “老板……小杨不懂啊。我先**看看**这个项目，我先去**学习学习**，我先去**研究研究**……”

于是小杨打开了那个 README——几千行、全是术语、图还挂在外链加载不出来；
打开那个组织——十几个仓库，不知道哪个是主角、该先看哪个；
打开那篇 arXiv 和那个领域——几十篇论文、博客、视频，从何下手都不知道。
**学是要学的，但只靠硬啃，又慢又枯燥，还容易抓不住重点。**

**如何快速摸清一个不太熟悉的领域或内容？** 这套 skill 就是为此而生——
让 AI 替你把它先**嚼碎、核实、整理**成一份循序渐进的学习材料，
你打开就能学，而不是从一堆原始资料里大海捞针。

## 它能帮小杨做什么

给它一个来源，它替你完成「调研 → 梳理 → 生成」，产出一个**双击即开、可离线、可分享**的学习网页：左边目录、右边内容，把来源自己的架构图、截图、视频封面都下载到本地，再配上术语表、代码结构导读、可交互的学习内容。

**定位（也是它的边界）：**

- 🚀 **为「快速学习、快速摸清」而生**——先看懂、先上手某个项目 / 领域 / 论文 / 书，而不是写一本权威手册。
- 👤 **个人学习优先**——给自己用的学习材料，不是团队 wiki、不是文档站、不是知识库系统。
- 📄 **产物是单个静态 HTML，可分享**——自包含、可离线、双击即开，能直接发给别人；没有构建、没有服务器、没有依赖。
- 🧩 **可单独用，也可合并用**——每个 skill 自包含、能独立安装独立触发；想要一个总入口，就用 `/learn` 自动判型分流。
- 🪶 **刻意轻量**——一次抓取整理 + 一个静态页，够用就好；要更新就重跑一遍，不堆重型基建。

整个家族**风格统一、交互一致、且坚持「真实不编造」**——不管你学什么，体验都是同一套，数据都来自真实抓取。

| Skill | 学习的来源 | 状态 |
|-------|-----------|------|
| **[github-project-learn](skills/github-project-learn)** | 一个 GitHub 仓库或整个组织 | ✅ 已完成 |
| **[domain-learn](skills/domain-learn)** | 一个开放领域（如 “3DGS”“扩散模型”），跨网络调研成「入门→进阶→实践」路线，含可交互 demo | ✅ 已完成 |
| **[textbook-learn](skills/textbook-learn)** | 一本技术书 / PDF，按章拆成课程，配例题精讲与主动回忆测验，带进度追踪 | ✅ 已完成 |
| **[paper-learn](skills/paper-learn)** | 一篇论文 / arXiv，逐图拆方法 + 公式离线渲染 + 可交互机制 demo + 「已证实 vs 只是假设」批判性阅读 | ✅ 已完成 |

不想记四个名字？一个总入口 **[`/learn`](skills/learn)** 就够了——丢给它任何来源（GitHub 链接 / arXiv / 一本书 / 一个领域），它**自动判型**分流到上面对应的 skill；模糊时先问一句再分流。（页攒多了之后，它还能把所有学习页[聚合成一个离线书架](samples/learn-hub/index.html)——这是个可选附属功能，平时用不到。）

每个 skill **独立自包含、可单独安装**——用其中一个，不需要整个仓库。

**单仓库** 与 **整个组织**，自动走不同的页面结构：

| 单个项目（如 OmAgent） | 整个组织（如 QwenLM） |
|---|---|
| ![](docs/images/01-overview-omagent.png) | ![](docs/images/02-org-mode-qwen.png) |
| 直接深入：是什么 / 架构 / 原理 / **代码结构** / 部署 | 多一层：**全景选型** +「**重点仓库逐个详解**」+ 旗舰深入 |

组织模式到底有多完整？这是 QwenLM 那页**从头滚到底的全貌**——速览、全景选型表（按真实星标排序）、「你想干什么先学哪个」的选型地图，再到逐个详解：

![QwenLM 组织模式完整长图](docs/images/04-qwen-tall.png)

每个页面内置的交互：吸顶目录 + 滚动高亮、图片点击放大、代码一键复制、术语表实时搜索。

想直接上手看真实产出？[`samples/`](samples/) 里有可离线打开的完整案例（双击 `index.html` 即可）。

## 产出长什么样

一个无需构建、无需服务器、完全离线就能打开的文件夹：

```
<thing>-learn/
├── index.html      # 自包含(内联 CSS+JS)，双击即开
├── assets/         # 来源真实的图片 / 架构图 / 媒体，已下载到本地
└── scripts/        # 可选的辅助脚本（如环境体检 / 一键安装，默认 dry-run）
```

## 安装

**方式 A — 打包好的 `.skill` 文件**（从 Release 下载）：拖进 Claude Code 的 skill 安装器即可。

**方式 B — 从源码安装**：把某个 skill 文件夹拷进你的 skills 目录：

```bash
cp -r skills/github-project-learn ~/.claude/skills/
```

然后对 Claude 说一句，比如 *“帮我学习 github.com/openai/whisper”*，skill 就会自动触发。

## 仓库结构

```
ai-learning-skills/
├── skills/                     # 每个子目录 = 一个可安装的 skill
│   └── github-project-learn/
├── shared/                     # 跨 skill 复用的「设计内核」
│   ├── template.html           # 页面外壳（CSS + 交互 JS）
│   ├── learning-page-design.md # 设计系统说明
│   └── fetch-media.sh          # 健壮的媒体下载器（识别 LFS / 走代理绕墙）
└── docs/
    └── adding-a-new-skill.md    # 加下一个 skill 的约定
```

**关于 `shared/`：** 因为 skill 必须自包含才能单独安装，所以每个 skill 都**自带一份**
共享模板/脚本的拷贝；`shared/` 是**单一事实来源**——改进设计系统时在这里改，再同步回各
skill。详见 [docs/adding-a-new-skill.md](docs/adding-a-new-skill.md)。

## 原则

- **真实，绝不编造。** 星标、日期、媒体链接、跑分——全部来自真实抓取；不存在的东西（比如某项目没有教学视频）页面会如实说明。
- **离线优先。** 媒体一律下载到本地，页面双击即开、便于分享。
- **设计一致。** 整个家族共用一套外壳和交互。
- **重原始，轻转述。** 架构图、真实代码结构、一手资料，胜过对 README 的复述。
